Performance

To get the best performance out of this template use lazy loading and have no more than 50 items in a single page of a gallery. For galleries with more than 50 items enable paging and use one of the built-in paging types with a page size set to 50.

If needed the number of items could be extended up to around 100 per page, but once you get up that high Lighthouse tests will start reporting excessive DOM size despite performance still being good.

Large Galleries and/or Paged Galleries

A large gallery is defined as any gallery that has more than 100 items. A paged gallery is defined as any gallery that has paging enabled.

Recommended

HTML + JSON Items + Lazy Loading

The first pages items should be delivered as HTML directly from the server, any additional items should be provided through JSON and created by the front-end when required.
